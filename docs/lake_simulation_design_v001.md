# Lake Simulation Design v001

## コンセプト

見た目はSFC風の親しみやすい2D釣りゲーム。

内部では湖全体をリアルタイムにシミュレーションし、
プレイヤーは魚そのものではなく「湖の状態」を読みながら釣果を伸ばしていく。

本作のコンセプトは、

「見た目は1995年のSFC、内部は2026年の釣りシミュレーター」

である。

プレイヤーは以下を考えながら釣りを行う。

- 水温
- 天候
- 時間帯
- 風向き
- ベイトの位置
- 地形変化
- 水深
- ストラクチャー

# 1タイル1ビューシステム

本作の特徴。

全ての岸タイルは個別の釣りビューを持つ。

同じ湖でも場所によって

- 見える景色
- 水深
- 障害物
- ベイト量
- 魚の付き場

が異なる。

```js
viewData = {
  background: "rock_bank",

  leftObject: "reed",

  centerObject: "none",

  rightObject: "fallen_tree",

  visibleDepth: true
}
```

# 水中地形

## シャロー

浅場。

春や朝夕に魚が入りやすい。

---

## フラット

変化の少ない平坦な地形。

---

## ブレイクライン

急激な水深変化。

最重要地形の一つ。

---

## チャネル

水中の溝。

魚やベイトの回遊ルート。

---

## 岩場

甲殻類やベイトが集まりやすい。

---

## 沈み木

大型バスの好ポイント。

根掛かりリスクあり。

---

## ウィード

隠れ場所。

魚の待機場所になる。

# 魚の行動

魚は決められたルートで動かない。

周囲環境を評価して移動する。

評価要素

- 水温
- 酸素量
- ベイト量
- カバー
- プレッシャー
- 水深

---

## 行動状態

- 待機
- 回遊
- 捕食
- 追尾
- 警戒
- 逃走

# プレッシャーシステム

同じ場所に投げ続けると上昇。

```js

```

影響

- 見切りやすくなる
- 追尾距離低下
- バイト率低下
