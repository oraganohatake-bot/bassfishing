# NPC Information Design v001

## コンセプト

本作では魚探や攻略サイトではなく、

「人から得た情報」

が重要な情報源となる。

プレイヤーは湖を探索しながら、

- 地元アングラー
- 常連客
- 釣具店店主
- 大会参加者

などと会話し、

湖について学んでいく。

情報は常に正しいとは限らない。

プレイヤーは情報を集め、
検証し、
自分だけの知識を構築していく。

---

# 情報システムの目的

本作はデータを見るゲームではない。

プレイヤーが

```text
会話
↓
仮説
↓
検証
↓
経験
```

を繰り返すことで成長する。

# 設計方針

プレイヤーに全ての情報を与えない。

景色を見て、

人と話し、

経験を積み、

湖を理解していく。

本作におけるNPCは、
クエスト発行機ではなく、

「湖の知識を伝える存在」

である。
