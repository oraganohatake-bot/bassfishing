Bass RPG  –  Beta v0.9

2D リアル系バス釣りシミュレーター。  
SFC 風トップダウン探索マップ＋ 1 人称釣りビューの 2 画面構成。

## 起動方法

### 必要環境
- Python 3.9+
- pygame 2.x

### インストール

```bash
pip install pygame
```

### 起動

```bash
cd bass_rpg
python3 main.py
```

## 操作方法

### 探索マップ（トップダウン）

| キー | 操作 |
|------|------|
| `W` / `↑` | 北へ移動 |
| `S` / `↓` | 南へ移動 |
| `A` / `←` | 西へ移動 |
| `D` / `→` | 東へ移動 |
| `E` | NPC に話しかける / 近くの釣りポイントで釣りを開始 |
| `F2` | **大型魚一覧**（個体管理デバッグ）ON/OFF |
| `F3` | **NPC一覧**（友好度・知識デバッグ）ON/OFF |
| `F4` | **大型魚テストモード** ON/OFF（次回の釣りで 52/58/64cm を追加スポーン） |
| `F5` | **手動セーブ**（"Saved!" と表示） |
| `F9` | **ロード**（セーブファイルから復元） |
| `ESC` | ゲーム終了 |

> 画面内の黄色い丸マーカーが釣りポイントです。  
> 近づくと下部に「Press E → ポイント名」と表示されます。
>
> **NPC**（色付き丸）に近づくと「Talk [E] → NPC名」が表示され、`E` で会話できます。  
> NPCが近くにいる場合は NPC 会話が優先されます（釣りポイントより近いと釣り開始に使えない）。

### 釣りビュー（1 人称風）— Beta v0.9 操作系

> **操作系**:  
> **キャスト前 = 十字キーで狙う・A/D で足場移動 / 着水後 = 十字キーでロッド・マウスでリール**
>
> 十字キーは状態で役割が切り替わる：**キャスト前 = キャストカーソル移動**、**着水後・ファイト中 = ロッド操作**。

**キャスト前（FS_IDLE）**

| 操作 | 内容 |
|------|------|
| `←` `→` `↑` `↓` | **キャストカーソル移動**（水面の黄色い十字を動かして狙う） |
| `A` / `D` | **足場移動**（プレイヤーの立ち位置を左右へ。立ち位置でアプローチ角が決まる） |
| `LMB` 長押し | **キャスト溜め開始**（振りかぶり） |
| `LMB` 離す | **リリース**（カーソル位置へ投げる。ゲージ位置で PERFECT / GOOD / EARLY 判定） |
| `1` 〜 `6` | **ルアー切替** |

**着水後（FS_RETRIEVE / FS_BITE / FS_FIGHT）— 足場は固定**

| 操作 | 内容 |
|------|------|
| `LMB` 長押し | **通常リーリング** |
| `LMB` 短押しタップ | **チョイ巻き**（ワーム/ジグの微調整） |
| `LMB` 連打 | **早巻き**（リアクション狙い・高速回収） |
| `↓` 短押し | **トゥイッチ**（小さくルアーが跳ねる）★バイト率大幅アップ |
| `↓` 長押し | **リフト**（ロッドを立てて持ち上げる / テンション増加） |
| `↑` | **フォール**（ロッドを下げてテンションを抜く / 沈める） |
| `←` `→` | **ライン角度調整**（コース微調整 / ファイト時の魚制御） |
| 何も押さない | **ストップ**（操作直後の中立 = 食わせの間）★最も食わせやすい |
| `↓`（アタリ中） | **フッキング**（`SPACE` も補助として使用可） |
| `↓` 長押し（ライン長 1.5m 以内） | **ランディング** |
| `K` / `R` | **キープ / リリース** |

**共通**

| 操作 | 内容 |
|------|------|
| `F2` | デバッグ表示 ON/OFF（旧 `D`。`A/D` 足場移動と衝突のため移設） |
| `F4` | 大型魚テストモード |
| `ESC` | 探索マップへ戻る（ファイト中はギブアップ） |

> **足場移動はキャスト前のみ**。着水後・ファイト中は移動不可、回収完了で再び移動可能。

**釣りの流れ（Beta v0.9）**

```
LMB 押下 → キャストゲージ上昇 → GREEN ゾーンで離す（PERFECT!）
  ↓
着水 → LMB長押しで巻く / ↓トゥイッチ → 止めて食わせの間
  ↓
魚が CHASE → Bite Window チャージ → バイト発生!
  ↓
ルアー別フッキング:
  クランク/スピナベ → 自動（ゴン!）
  ワーム/ジグ      → ティックを感じたら一呼吸置いて ↓
  トップ           → バシャ! の後、重みが乗ってから ↓
  ↓
50cm未満 → 即キャッチ / 50cm以上 → ファイト突入!
  ↓
テンションゲージを GREEN〜YELLOW に保ち、走られたら ↑ で逃がす
  ↓
1.5m まで寄せて ↓ 長押し → ランディング!（最後の突っ込みに注意）
  ↓
[K] KEEP  または  [R] RELEASE
```

## キャストシステム（Beta v0.9）

LMB 押下でゲージが上昇（約1.25秒で満タン）。離したタイミングで判定。

| 判定 | ゲージ位置 | 着弾誤差 | 備考 |
|------|----------|---------|------|
| **PERFECT** | 78〜92% (緑帯) | ±0〜0.5セル | **大型魚(40cm+)反応率 +50%** |
| GOOD | 60〜78% / 92〜100% | ±1〜2セル | |
| EARLY | 60%未満 | ±1〜2セル | ショートキャスト（手前に落ちる） |
| LATE | 100%超 / 限界保持 | ±2〜4セル | オーバーキャスト（奥にずれる） |

## バイト＆フッキング（Beta v0.9）

ルアーごとにアタリの出方とフッキング方式が異なります。

| ルアー | アタリ演出 | フッキング方式 |
|--------|-----------|--------------|
| Worm | LIGHT_TICK（…tick） | **DELAY** — 一呼吸置いて `↓`（早合わせはPOOR） |
| Jig | MEDIUM_TICK（TICK!） | **DELAY** — 同上 |
| Crankbait | HEAVY_STRIKE（ゴン!） | **AUTO** — 自動で掛かる |
| Spinnerbait | HEAVY_STRIKE | **AUTO** — 自動で掛かる |
| Minnow | MEDIUM_TICK | **HYBRID** — 50%自動 / 50%手動 |
| Topwater | SPLASH!（バシャ!） | **VISUAL_DELAY** — 重みが乗るまで待って `↓`（早合わせは50%すっぽ抜け） |

DELAY系のフッキング品質（`↓` を押すタイミング）:

```
早すぎ(15f未満) = POOR → 30f前後〜90f = JUST（最良） → 遅れるほど低下
```

フック品質は hook_hold 初期値に直結: JUST=100 / GOOD=75 / NORMAL=55 / POOR=35

## ファイトシステム（Beta v0.9）

**50cm 以上**の魚を掛けるとファイトが発生します（テンション管理ゲーム）。

### 内部値

| 値 | 内容 |
|----|------|
| `hook_hold` | フックの保持力。0 でフックアウト。緩めすぎると減る |
| `fish_stamina` | 魚の体力（**非表示**）。プレッシャーを掛けると減る |

### テンションゲージ（画面下・横長）

```
[ 青 BLUE | 緑 GREEN | 黄 YELLOW | 赤 RED ]
   ←緩い        理想       プレッシャー   危険→
```

| ゾーン | 効果 |
|--------|------|
| **BLUE** (緩み) | hook_hold 減少 → フックアウト危険 |
| **GREEN** (理想) | hook_hold 維持・魚を制御可能 |
| **YELLOW** | fish_stamina 減少（魚が疲れる） |
| **RED** (危険) | スタミナ大幅減だが、**ラインブレイク / フックアウトのリスク** |

### ファイト操作

| 状況 | 正解操作 |
|------|---------|
| 魚が走る（RUN!） | `↑` でテンションを抜いて走らせる + 魚の進行方向と**逆**へ `←`/`→` |
| 通常時 | LMB で巻いて寄せる（GREEN〜YELLOW 維持） |
| RED に入った | `↑` でテンションを逃がす（REDで巻くと危険） |
| 障害物に潜られた（IN COVER!） | 障害物と逆方向へロッドを倒して引き剥がす |
| 魚が 1.5m 以内 | `↓` 長押しでランディング |

> **魚の頭の向き**は水面の黄色い矢印で表示されます。進行方向と逆へロッドを倒すと制御しやすくなります。  
> ランディング直前、元気が残っている魚は**最後の突っ込み（LAST DASH!）**をします。

### サイズ別の手強さ

| サイズ | 特徴 |
|--------|------|
| 〜50cm | ファイトなし（即キャッチ） |
| 50cm〜 | ダッシュあり |
| 60cm〜 | 方向転換が頻繁 |
| 65cm+ | 複数回の長距離ラン |
| レジェンド級 | 停止 → 急反転 → ダッシュの特殊行動 |

### ロッド描画

ロッドは画面右下に動的描画され、**しなり = テンション**を視覚的に伝えます。
テンション 90% 超でティップが赤く点滅 = ラインブレイク警告。

> **コツ①**  魚が CHASE になったら巻くのを止める（ストップ）か `↓` 短押し（トゥイッチ）。  
> **コツ②**  同じアクションを長く続けると Naturalness が下がり、魚がスプークします。  
> **コツ③**  PERFECT キャストでピンスポットに落とすと大型魚の反応率が大幅アップ。  
> **コツ④**  ファイトでは焦って巻かない。RUN 中は走らせ、止まったら巻く。

## Beta v0.95 整合性修正（操作・物理の違和感対応）

v0.95 確認中に出た操作・物理挙動の違和感を、新機能追加ではなくライン/ロッド/ルアー回収・ファイト・バイトの**整合性修正**として対応。

### プレイヤーの立ち位置・足場移動・キャストカーソル・短いロッド

「岸から狙って投げる」感覚を強めるため、ロッドを画面内UIに見せず、**足場移動＋十字キーキャスト**で角度を作る方式。

- **ロッドスタンスのキー切替（旧 Z/X/C）は廃止**。ライン角度はプレイヤーの**立ち位置（足場）** で作る。
- `FishingView.player_stance_x`（0.0=左の岸〜1.0=右の岸）。釣りビュー入室時に岸位置から決定（現状 spot/seed 由来）。
- **足場移動（A/D）はキャスト前のみ**。着水後・ファイト中は移動不可、回収完了で再び可能。移動範囲 `STANCE_MIN`〜`STANCE_MAX`（0.20〜0.80）、速度 `STANCE_MOVE_SPEED`。
- **ロッドのバット位置（手元）は `player_stance_x` に連動**し、画面下端のさらに下（`ROD_BASE_Y`）から伸びる。ティップもそこから計算。ライン描画は `lure → rod_tip` なので、立ち位置が変われば**同じ着水点でもアプローチ角が変わる**。
- **ロッド表示長は短く**（`ROD_VISUAL_LENGTH`）。画面下から少し出る程度に抑え、ロッドはライン/テンションの補助として見せる（ファイトのしなりは `rod_flex` で別途表現）。水面に長く被らない。
- **キャストポイントは十字キーで選択**。`cast_cursor_x / cast_cursor_y`（速度 `CAST_CURSOR_SPEED`）を十字キーで動かし、水面に黄色い十字で表示。`LMB` 長押しで溜め → 離してカーソル位置へキャスト（マウス位置は使わない）。
- **リトリーブはロッドティップへ急吸収させず**、`player_stance_x` 基準の自然な回収ラインへ向かう（`_retrieve_target_cell_x()` → `Lure.retrieve_target_x`）。回収は y 方向（手前へ戻す）が主、x は微調整のみ：`approach = clamp(near_factor × 0.02, 0, 0.035)`、横移動量は `±0.03 cell/frame` で頭打ち。一直線の横スライドにならない。
- 調整値（`tuning.py`）：`STANCE_MOVE_SPEED` / `STANCE_MIN` / `STANCE_MAX` / `CAST_CURSOR_SPEED` / `ROD_VISUAL_LENGTH`。

### ライン長管理（`line_length_m`）— v0.95で2D化

- ファイト中の `line_length_m` は **魚のグリッド座標 → プレイヤー立ち位置（アンカー）の2D斜距離 × `FIGHT_METERS_PER_CELL`** で常に再計算される（`FightState.line_length_m`）。**画面Y/縦距離だけでは決まらない**。
- アンカー＝立ち位置：`(player_stance_x × (UW_SIZE−1), UW_SIZE−1)`（手前端＝岸）。`FishingView._player_anchor_cell()`。
- **横に大きく走った魚は、同じYでも `line_length_m` が長くなる**（斜距離が伸びるため）。
- **初期ライン長 = フッキング地点（ルアー位置）からアンカーまでの2D距離**（`start_cell=(lure.x, lure.y)`）。奥／横で掛けるほど長く、寄せに時間がかかる。
- **リールを巻くと魚がアンカー方向へ寄り**（`line_length_m` 減）、巻かない／走られると `line_length_m` が増える。
- **ランディング条件は2Dの `line_length_m <= 1.5m`**（`FIGHT_LANDING_DIST_M`）。画面位置だけでは判定しない。ファイトパネルは `Line out: X.X m` を表示。

### バイトトリガー改訂（イベント駆動）

- 「魚が近くにいるからゲージが溜まる」だけの挙動を弱め（`BITE_PASSIVE_SCALE`）、実釣的な**イベントの瞬間に `bite_check`** を走らせる方式へ。
- トリガーイベント：**ルアー停止 / フォール開始 / ロッドアクション（トゥイッチ・リフト）直後 / ストラクチャ通過直後 / 魚がルアーに追いついた瞬間**。
- 各イベントで 1 回だけ確率判定（`BITE_EVENT_BASE_P × イベント別重み × 環境 × ルアー適合 × 記憶 × キャスト`）。連続発火は `BITE_EVENT_COOLDOWN` で抑制。
- フッキング方式は v0.95 を踏襲：DELAY系（ワーム/ジグ）は違和感表示→一拍待って↓で合わせる。**AUTO系（クランク/スピナベ）は ↓許容窓を広めに取った準自動**（`AUTO_JUST_END` / `AUTO_GOOD_END` / `AUTO_TIMEOUT`）。

新パラメータは全て `tuning.py`（`FIGHT_METERS_PER_CELL` / `FIGHT_START_LEN_MIN/MAX` / `FIGHT_VISUAL_RANGE_M` / `BITE_EVENT_BASE_P` / `BITE_EVENT_WEIGHTS` / `BITE_PASSIVE_SCALE` / `BITE_EVENT_COOLDOWN`）。C&R・個体管理・セーブ・環境・F4テストモードは変更なし。

## Beta v0.95 追加修正（キャスト飛行・ロッドリセット・魚の遊泳）

### キャスト飛行演出（`FS_CASTING`）

- これまで着水点に**ルアーが突然出現**していたのを、**手元（ロッドティップ）から着水点へ放物線で飛ぶ**演出に変更。
- フロー：LMB リリースでキャスト確定 → `FS_CASTING`（飛行）→ 着水で `lure.cast()` ＋ 波紋発生 → `FS_RETRIEVE`。
- 飛行中は **`lure.in_water = False`**（まだ水中判定は始まらない）。着水して初めてリトリーブ・バイト判定が動く。
- 飛行時間は距離に応じて **20〜45 frame**、放物線の高さは遠投ほど高い。薄いライン軌跡＋小さなルアー点＋ティップ→ルアーの放出ラインを描画。
- キャスト精度（PERFECT/GOOD/EARLY 判定・着水点のばらつき）は従来どおり維持（飛行は演出のみで、着水セルは確定済み）。
- 内部値：`_cast_flight_start / _cast_flight_target / _cast_flight_cell / _cast_flight_timer / _cast_flight_duration / _cast_arc_height`。

### 釣果後のロッドリセット

- `RodController.reset()` が **`rod_visual_x / rod_visual_y`（描画上のしなり角）も 0 に戻す**ように修正（従来は入力値のみリセットで、前回ファイトの角度が残っていた）。
- リセットを呼ぶタイミング：**ランディング成功 / KEEP・RELEASE後（即キャッチ含む）/ バラシ・ギブアップ / すっぽ抜け / FS_IDLE 復帰 / ESC で釣りビューを抜ける / 次キャスト確定時**。
- 結果：釣果後・次キャスト時にロッドが必ず中立角度へ戻る。

### ファイト時の魚の遊泳（2Dグリッド移動）

- 魚は **グリッド座標 `fish_x / fish_y` で2D移動**する。基準ベクトルは `off`＝アンカー(立ち位置)→魚（沖方向）／`-off`＝魚→アンカー（寄せ方向）。`head_dir` は横成分（左右の走り）。
  - **RUN**：沖＋横へ走る（横成分＝`head_dir × FIGHT_LATERAL_RUN_SPEED × run_power`）。
  - **DIVE**：沖方向強め（`FIGHT_DIVE_OUTWARD`）。
  - **SHAKE**：小刻み左右。**TURN**：`head_dir` 反転。グリッド端で自然に向き変更。
- **リールを巻くと魚はアンカー（立ち位置）方向へ収束**する：立ち位置が左なら左手前へ、右なら右手前へ寄る（`FIGHT_REEL_GAIN_BASE × FIGHT_REEL_PULL_MULT`、巻き中は横走り `lat` を減衰させて収束を明確化）。
- **巻かないと沖へ逃げて `line_length_m` が増える**：RUN/DIVE に加え、**リールを巻いていない間は魚が常にじわっと沖へドリフト**（`FIGHT_IDLE_DRIFT`、高テンションで抑制＝張れば止められる）。
- サイズ補正（`FIGHT_SIZE_AI` の `run_power`）：**40cm=小さく / 50cm=明確に左右へ走る / 60cm=大きく走る / 65cm+=急反転・沖走り**。
- **表示**：魚影・ライン終端は `_uw_to_screen(fish_x, fish_y)` で2D位置をそのまま描画。**立ち位置によってティップ→魚のライン角度が変わり、立ち位置方向へ引いているのが見える**。
- 新パラメータ：`tuning.py` の `FIGHT_DIVE_OUTWARD` / `FIGHT_IDLE_DRIFT`。寄せ速度は v0.95 でリール係数化（下記参照）。C&R・個体管理・KEEP/RELEASE・F4テストモードは変更なし。

## Beta v0.95 フィーリング微調整（キャスト高速化・リーリング減速）

キャストをキビキビさせ、リーリングを現実寄りに遅くする調整。あわせて**将来のリール性能差**に備え、巻き取り速度を固定値からリール係数（`ReelSpec`）由来へ置き換えた。

### 1. キャスティングを高速化

- キャストゲージ最大到達時間を **1.0秒 → 約0.67秒**（`CAST_CHARGE_RATE = 100/40`、約33%速）。
- 「振りかぶる→ロッドの弾性で離す」の1秒未満のスピード感。即時すぎて反応ゲーにならない程度に留める。
- 高速化で入力窓が縮むぶん **PERFECT帯を 76–93 → 72–93 に拡大**（窓は約140ms＝8.4f）。短縮後も PERFECT を狙える。
- `CAST_GOOD_LO` も 60 → 56 に下げ、GOOD の取りこぼしを緩和。

### 2. リーリングを減速（現実寄り）

- 通常リトリーブ速度を **約0.45 m/s** に大幅減速（旧 ≒1.9 m/s）。序盤タックルの「1秒0.5m」イメージ。
- 早巻き倍率も **1.8 → 1.5**（早巻きでも約0.68 m/s に抑制）。
- ファイト中の寄せ速度も同基準へ：元気な魚 ≒0.40 m/s、バテた魚 ≒0.63 m/s（疲労ボーナス `FIGHT_REEL_FATIGUE_BONUS`）。
- 結果、`line_length_m` の減少が現実寄りになり、大型魚ファイトに寄せの「間」が生まれる。

### 3. リール性能の将来拡張（`ReelSpec`）

巻き取り速度を直接固定値で持たず、リール1個分の性能 `ReelSpec` から導出するようにした。装備システム未実装のため **Beginner Reel 固定**（`TU.EQUIPPED_REEL`）。

```python
ReelSpec(
    name="Beginner Reel",
    retrieve_speed_mps=0.45,  # リトリーブ/ファイト時の基準巻き取り速度
    drag_power=0.45,          # 高テンション時に耐えられる力
    line_control=0.40,        # テンション変化の安定性
)
```

| 係数 | 使い道 | 現状の参照 |
|------|--------|-----------|
| `retrieve_speed_mps` | リトリーブ／ファイト寄せ速度の基準（m/s → セル/f 換算） | `REEL_RETRIEVE_SPEED`, fight gain |
| `drag_power` | 高テンション時に耐えられる力 | 未配線（将来 `FIGHT_RED_STRESS_*` 等へ） |
| `line_control` | テンション変化の安定性 | 未配線（将来テンション揺れ幅へ） |

> m/s → セル/フレーム換算は `mps_to_cells_per_frame()`。リトリーブは `REEL_CELL_M`（水中グリッド1セル≒0.5m）、ファイトは `FIGHT_METERS_PER_CELL` を基準にする。

### v0.95 確認項目

1. キャスト最大溜めが1秒未満（約0.67秒）✓
2. PERFECT が狙える（窓 ≒140ms）✓
3. 通常リーリングが遅い（≒0.45 m/s）✓
4. 早巻きでも速すぎない（≒0.68 m/s）✓
5. ファイト中に巻いても魚が一瞬で寄らない（≒0.4〜0.6 m/s）✓
6. `line_length_m` の減少速度が現実寄り ✓
7. F4大型魚ファイトが長すぎず短すぎず → 実プレイで要確認（`FIGHT_STAMINA_PER_CM` / `EQUIPPED_REEL.retrieve_speed_mps` で調整）

### バランス検証結果（シミュレーション30回試行）

## 魚の反応段階（Phase 3）

魚はルアーに対して以下の段階で反応します。右サイドバーのドット色と、画面下部の「FISH REACTION」パネルで確認できます。

| ステージ | 色 | 意味 | 移行条件 |
|---------|-----|------|---------|
| ignore | グレー | ルアーに無関心、ポイント周回中 | — |
| notice | 黄色 | ルアーを発見、注視中 | 検知範囲内 + 深度一致 + appeal ≥ 0.22 |
| approach | 橙 | 興味を持ち近づいている | notice 後、appeal ≥ 0.38 |
| CHASE | 黄橙 | 追尾コミット・高速接近 | approach 後、距離3.5セル以内 |
| BITE! | 赤 | 射程内でバイト発生 | Bite Window フル充填 |
| spooked | 紫 | 見切り・逃走中 | 速すぎ / 不自然 / 深度ミス |

### 見切り（スプーク）の主な原因

| 原因 | 閾値 | 発生タイミング |
|------|------|--------------|
| ルアーが速すぎる | speed > 0.11 セル/frame | approach 中、活性 < 0.65 の時 |
| 不自然な動き | naturalness < 0.22 | chase 中 |
| 深度ミスマッチ | 差 > 2.4m | approach 中 |
| セルプレッシャー高 | pressure > 8 | approach→chase 移行時 |

## Catch & Release システム（Phase 10）

### 設計思想

魚は消費リソースではなく、湖で生き続ける個体。  
プレイヤーは釣った後に「持ち帰る」か「湖に返す」かを選択できる。  
リリースした魚は成長し、学習し、将来また釣れる可能性がある。

> **「去年リリースした魚と再会した」という体験を作ること**が本作の最大の目標。

---

### KEEP / RELEASE 選択

魚をフッキング後、以下のオーバーレイが表示されます：

```
★ RECAPTURE ★        ← 再捕獲時のみ表示
Last caught: Day 3
Was: 52.4cm → Now: 53.2cm

[B51003]
53.2cm
Age 7
Released 4x

[K] KEEP   + 532 pt
[R] RELEASE
```

| 選択 | 処理 |
|------|------|
| **K** (KEEP) | 個体削除・報酬獲得（`length_cm × 10 pt`）・釣果ログに KEEP 記録 |
| **R** (RELEASE) | 個体維持・health −0.05・caution +0.05・リリース回数加算・釣果ログに RELEASE 記録 |

---

### 報酬計算

```
reward = int(length_cm × 10)
```

## Beta v0.95 ファイト調整（やりとり感の強化）

ファイト中に「魚とやりとりしている感」を強めるための調整。**新グラフィックではなく挙動で**伝える。

### 1. 魚の逃走アクションを増やす

- RUN / DIVE の基礎発生率を引き上げ（`FIGHT_RUN_P_BASE` 0.45→0.60 / `FIGHT_DIVE_P_BASE` 0.20→0.28）。
- 発生率は `× stamina_ratio × run_power(サイズ)` なので、**大型・高スタミナほど頻繁に沖へ走る**。
- 40UP は沖へ逃げ、50UP は複数回走り、60UP は長距離ランを繰り返す（`FIGHT_SIZE_AI` の `run_len` をサイズで拡大）。

### 2. 逆方向ロッド操作で疲労を削る（`control_success`）

| 状況 | 正解入力 | 効果 |
|---|---|---|
| 魚が左へ走る | `→` | 制御成功 |
| 魚が右へ走る | `←` | 制御成功 |
| 魚が沖へ走る／潜る | `↓` | 浮かせる／止める＝制御成功 |
| 間違った方向へ倒す | 同方向 | 沖へ走られやすい（`wrong_dir`）|

- `fish_move_dir`（= `head_dir` の符号）と `rod_input_dir`（`rod_x`）の積が負なら **制御成功**。
- 制御成功中：`fish_stamina` が余分に減り（`FIGHT_CONTROL_STAMINA_DRAIN`）、沖へのライン放出が抑制される（`FIGHT_CONTROL_OUTWARD_SUPPRESS`＝**伸びが止まる手応え**）。
- 間違った方向：沖移動が増える（`FIGHT_WRONG_DIR_OUTWARD_MULT`）。

### 3. fish_stamina 低下による行動変化

スタミナが下がると（`> 70%` 元気 / `40–70%` 通常 / `15–40%` 鈍る / `< 15%` ランディング圏）：

- RUN / DIVE / 向き変更の頻度が低下（全て `× stamina_ratio`）。
- 移動速度・ライン放出が弱まり、リールで寄せやすくなる（疲労ボーナス `FIGHT_REEL_FATIGUE_BONUS`）。

### 4. サイズ別ファイト設計（`tuning.py` 集約）

`max_stamina = size × run_power × FIGHT_STAMINA_PER_CM` で**大型ほど非線形に持久力が増す**。
最適操作での実測（ヘッドレス40試行平均、開始18m相当）：

| サイズ | ファイト時間 | ラン回数 | 感触 |
|---|---|---|---|
| 〜33cm | 約30秒 | 1–2 | ほぼ巻くだけ |
| 34〜39 | 約40秒 | 3–4 | 少し抵抗 |
| 40〜49 | 約47–50秒 | 5 | やりとりが必要 |
| 50〜59 | 約59–63秒 | 5–6 | 複数回走る |
| 60UP | 約71秒 | 6 | 別格・長距離ランを繰り返す |

> 目標の「60UP=5分級」はスタミナ膨張でテンポを損なうため、**相対的な別格感**を優先。ステアを怠ると（reel only）ファイトはさらに長期化しバラシも増える＝ステアの価値が明確。F4 テストモードは 52/58/64cm を即スポーンして差を短時間で確認できる。

### 5. ランディング接続

- 元気な魚（`stamina_ratio` 高）ほど `line<=1.5m` でも最後の突っ込みが起きやすい（発生確率 = `stamina_ratio × FIGHT_FINAL_DASH_P_SCALE`）。
- `fish_stamina` が低いほどランディング成功率が上がる。

### 6. UI/フィードバック

数値表示は不要（挙動で伝える）。デバッグ（**F2**）時のみファイトパネルに `stam %` と `CTRL+/wrong` を表示。
制御成功中はパネルヒントが「Good! Rod against the fish」に変わる。

> C&R・KEEP/RELEASE・個体管理・セーブ・F4 テストモードは変更なし。
